是时候学习vue3了

适用日期：2021-7
一、使用vite新建Vue3应用
Vite requires Node.js version >=12.0.0.

对于Yarn玩家而言:
创建vue3项目

yarn create @vitejs/app
进入vue3项目

cd <项目名称>
其他

yarn install //安装依赖
yarn dev //启动
yarn run build //发行

其他问题：
引入.vue时

在项目目录下添加vue-shims.d.ts文件，内容如下
declare module ‘*.vue’ {
import Vue from ‘vue’;
export default Vue
}

二、安装vue-router和vuex

yarn add vue-router@next vuex@next
在src目录下新建router/index.ts

在src目录下新建store/index.ts

三、CSS预处理器

yarn add less less-loader –dev

四、配置路径别名

yarn add @types/node –dev
vite.config.ts配置如下

import { defineConfig } from ‘vite’
import path from ‘path’
…
export default defineConfig({
…
resolve:{
alias:{
“@”:path.resolve(__dirname,”./src”)
}
}
…
})
别名配置后，你会发现import .ts后缀的文件时会有红色波浪线
An import path cannot end with a ‘.ts’ extension.
原来是import ts时，需要在tsconfig.json配置别名，且该文件中配置的别名需在vite.config.ts中对应存在，具体配置如下：
tsconfig.json
“compilerOptions”: {
“paths”: {
“@/“: [“src/“],
“@view/“:[“src/views/“],
“@utils/“:[“src/utils/“],
“@api/“:[“src/api/“]
}
}
vite.config.ts
resolve:{
alias:{
“@”: path.resolve(__dirname, “./src”),
“@css”: path.resolve(__dirname, “./src/assets/css”),
“@c”: path.resolve(__dirname, “./src/components”),
“@img”: path.resolve(__dirname, “./src/assets/img”),
“@style” : path.resolve(__dirname, “./src/style”),
“@view”: path.resolve(__dirname, “./src/views”),
“@utils”:path.resolve(__dirname, “./src/utils”),
“@api”:path.resolve(__dirname, “./src/api”),
}
},

五、配置服务器
vite.config.ts配置如下

export default defineConfig({
…
server:{
port:3000,
proxy:{
‘/api’:{
target:”http://localhost:8080",
changeOrigin:true,
rewrite: path => path.replace(/^/api/, ‘’)
}
}
}
…
})
target是你需要代理的地址，比如你的请求地址是
http://localhost:8080/product/list
那么target里面应该这么写：
target:’http://localhost:8080/'
编写请求的地方：
import { request } from ‘@/utils/request’
//测试请求
function getProductList(parameter){
return request({
method:’get’,
url:’api/product/list’,
parameter:parameter
})
最后上述配置会将你的url去掉api然后拼接上target去请求服务器

七、配置env
vite中的env配置，文档中有讲，这里从简总结。
首先在项目根目录（src同级）处新建.env.production和.env.development
然后在这2个文件中填写你需要的环境变量，用key=value的方式填写，换行区分。这里在两个文件中分别配置NODE_ENV=’development’和NODE_ENV=’production’
vite会根据你通过package.json中的哪个script启动项目，来确定项目的启动方式（MODE），然后通过MODE确定加载对应.env.[mode]文件的环境变量。
我们可以通过全局变量import.meta.env.MODE获取当前项目的MODE
通常我们不会把配置变量写在.env.[mode]文件中，而是在src新建config文件夹，里面创建index.ts根据import.meta.env.MODE确定导出对应的配置变量对象。

八、配置axios
网传ts配置axios有两种方式
第一种是建一个request.ts类放在utils文件夹，在request.ts中创建axios实例，配置拦截器，然后直接export出一个axios实例。这种方式的配置，可以直接在api.ts中导入后request.ts后直接使用。
第二种也是建一个request.ts类放在utils文件夹，不同的是export出的并不是一个axios实例，而是一个带init方法的axios自定义类。
介绍第一种：

使用配置好的axios
product.ts
import request from ‘@/utils/axios’;
export function article (parameter: any) {
return request({
url:’api/product/list’,
method:’get’,
data:parameter
})
}

九、配置mock

yarn add mockjs
yarn add vite-plugin-mock -D
在src同级新建mock目录，在mock目录中新建test.ts作为mock数据文件

在utils目录新建mockProdServer.ts文件

在vite.config.ts中配置vite-plugin-mock
vite.config.ts
plugins: [
vue(),
viteMockServe({
mockPath:’./mock’,
supportTs:true,
watchFiles:true,
localEnabled:true,
prodEnabled:command !==’serve’ && prodMock,
// 这样可以控制关闭mock的时候不让mock打包到最终代码
injectCode: import { setupProdMockServer } from '@utils/mockProdServer'; setupProdMockServer();
})
]

在vite.config.ts中关闭代理服务器，就可以使用ts文件模拟的mock数据了。需要注意的是如果启动项目时出错

9:30:31 ├F10: AM┤ [vite:mock] mock reload error Error [ERR_STREAM_DESTROYED]: Cannot call write after a stream was destroyed
events.js:292
throw er; // Unhandled ‘error’ event
这是目前esbuild的bug（2021-4至2021-7），使用下面命令即可解决

node ./node_modules/esbuild/install.js
