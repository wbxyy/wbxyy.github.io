啰里啰唆，不看也罢系列

git，分布式版本控制软件，用仓库的概念管理文件，支持文件版本的提交和回滚，更好地支撑项目开发。

git 本地仓库，一个隐藏的文件夹 .git，里面存放着提交过的所有版本，每个版本有对应文件的完整快照。

git 每次提交会新增一个版本，修改过的文件会完整地保存在该版本中，称为 快照 。未被修改的则保存上一版本的引用。

git 管理的目录叫 工作区，该目录下的文件为 工作区的文件

工作区的文件有 4 种状态：未追踪、未修改、已修改、已暂存。操作本地仓库的最终目的，就是让所有文件处在 未修改 状态，即工作区文件和本地仓库保持一致。

Promise 的初体验

Promise 是 ES6 新增的规范，它是 JS 中进行异步编程的新解决方案，可以很好地解决异步编程中回调地狱问题，让代码更加优雅。

JS 中常见的异步编程，比如数据库 IO，文件 IO，网络请求 AJAX，定时器等。

当我们调用异步任务时，是无法准确得知函数何时返回结果，通过 JS 顺序语句也无法获取结果，这时我们必须给异步任务提供回调函数。

让异步任务在合适的时机帮我们调用回调函数并传入结果，以便我们在回调函数中获取结果值并执行后续操作，比如渲染 DOM，比如写入文件，比如发起新一轮的网络请求。
